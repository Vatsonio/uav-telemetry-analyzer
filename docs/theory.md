# Теоретичне обґрунтування математичних перетворень

## 1. Системи координат

### WGS-84 (World Geodetic System 1984)
Глобальна геодезична система координат, стандарт для GPS.
Положення задається трійкою (latitude, longitude, altitude):
- **Latitude** (φ) - кут від площини екватора, [-90°, +90°]
- **Longitude** (λ) - кут від Грінвічського меридіана, [-180°, +180°]
- **Altitude** (h) - висота над еліпсоїдом або середнім рівнем моря (AMSL)

WGS-84 моделює Землю як еліпсоїд обертання з параметрами:
- Велика піввісь: a = 6 378 137 м
- Стиснення: f = 1/298.257223563

### ENU (East-North-Up)
Локальна декартова система координат, зручна для аналізу польотів:
- **East** (E) - на схід від точки відліку
- **North** (N) - на північ
- **Up** (U) - вгору (протилежно до гравітації)

Початок координат - точка старту БПЛА. Всі відстані у метрах.

### Перетворення WGS-84 → ENU

Для малих відстаней (<10 км) використовується лінеаризація (дотична площина):

```
E = (λ - λ₀) · cos(φ₀) · π/180 · R
N = (φ - φ₀) · π/180 · R
U = h - h₀
```

де R = 6 371 000 м - середній радіус Землі, (φ₀, λ₀, h₀) - точка відліку.

**Точніший підхід** (для великих відстаней):
1. WGS-84 → ECEF (Earth-Centered Earth-Fixed):
   ```
   X = (N + h)·cos(φ)·cos(λ)
   Y = (N + h)·cos(φ)·sin(λ)
   Z = (N·(1-e²) + h)·sin(φ)
   ```
   де N = a/√(1 - e²·sin²(φ)) - радіус кривизни, e² = 2f - f²

2. ECEF → ENU через матрицю обертання:
   ```
   [E]   [-sin(λ₀)        cos(λ₀)         0      ] [X - X₀]
   [N] = [-sin(φ₀)cos(λ₀) -sin(φ₀)sin(λ₀) cos(φ₀)] [Y - Y₀]
   [U]   [ cos(φ₀)cos(λ₀)  cos(φ₀)sin(λ₀) sin(φ₀)] [Z - Z₀]
   ```

## 2. Формула Haversine

Обчислює відстань по великому колу між двома точками на сфері:

```
a = sin²(Δφ/2) + cos(φ₁)·cos(φ₂)·sin²(Δλ/2)
c = 2·atan2(√a, √(1-a))
d = R·c
```

**Чому Haversine, а не сферичний закон косинусів?**

Формула сферичного закону косинусів:
```
d = R·arccos(sin(φ₁)·sin(φ₂) + cos(φ₁)·cos(φ₂)·cos(Δλ))
```

Ця формула страждає від числових похибок для малих відстаней (cos → 1,
втрата значущих цифр). Haversine використовує sin²(x/2), що залишається
числово стабільним навіть для відстаней у кілька метрів.

## 3. Трапецієвидне інтегрування

### Метод
Наближене обчислення визначеного інтеграла сумою трапецій:

```
∫[a,b] f(t) dt ≈ Σᵢ [0.5·(f(tᵢ) + f(tᵢ₊₁))·(tᵢ₊₁ - tᵢ)]
```

### Застосування: Прискорення → Швидкість
```
v(t) = v₀ + ∫[0,t] a(τ) dτ
```

### Похибка
- **Локальна похибка**: O(h³) для одного кроку (h = Δt)
- **Глобальна похибка**: O(h²) для всього інтервалу
- При IMU з частотою 50 Гц (h = 0.02 с), локальна похибка ~O(10⁻⁵)

### Проблема дрейфу при інтегруванні IMU

**Зміщення нуля (bias):** Акселерометр має постійну похибку δa ≈ 0.01-0.1 м/с².
Після інтегрування: δv = δa·t (лінійний дрейф швидкості).
Після подвійного інтегрування: δx = 0.5·δa·t² (квадратичний дрейф позиції).

Приклад: при bias = 0.05 м/с² через 60 с маємо:
- Похибка швидкості: 3 м/с
- Похибка позиції: 90 м

Тому на практиці IMU-інтегрування використовують разом з GPS через
фільтр Калмана (EKF), який "скидає" накопичений дрейф.

## 4. Кватерніони vs Кути Ейлера

### Кути Ейлера (Roll, Pitch, Yaw)
Інтуїтивне представлення орієнтації через три послідовні обертання.

**Проблема Gimbal Lock:** при Pitch = ±90° осі Roll та Yaw стають
паралельними - втрачається один ступінь свободи. Математично:
```
R(ψ)·R(±90°)·R(φ) = R(ψ ± φ)·R(±90°)·I
```
Результат залежить лише від (ψ ± φ), а не від ψ та φ окремо.

### Кватерніони q = (w, x, y, z)
4-компонентне представлення обертання, де |q| = 1:
- Не страждає від gimbal lock
- Ефективна композиція обертань: q₁₂ = q₁ · q₂
- Плавна інтерполяція (SLERP)
- Чисельно стабільніші

**Матриця обертання з кватерніона:**
```
R = [1-2(y²+z²)  2(xy-wz)    2(xz+wy)  ]
    [2(xy+wz)    1-2(x²+z²)  2(yz-wx)  ]
    [2(xz-wy)    2(yz+wx)    1-2(x²+y²)]
```

Ardupilot внутрішньо використовує кватерніони для EKF (Extended Kalman Filter),
а кути Ейлера - лише для зручності відображення.

## 5. Body Frame vs NED/ENU Frame

IMU вимірює прискорення та кутові швидкості у **body frame** (система координат
апарату). Для отримання прискорень у NED/ENU потрібно обертання:

```
a_NED = R(q) · a_body
```

де R(q) - матриця обертання з body frame у NED, побудована з кватерніона
орієнтації.

Без цього обертання, інтегрування прискорень body frame дає швидкості
у системі координат, що обертається разом з апаратом, що не має
фізичного сенсу для навігації!
